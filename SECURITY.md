# Security policy

## Supported version

Security fixes are applied to the latest default branch.

## Reporting

Open a private GitHub security advisory for vulnerabilities that could expose credentials, private source data, or local files. Do not include real cookies, access tokens, or private posts in a public issue.

## Design boundaries

This project intentionally does not store platform credentials, bypass access controls, or publish automatically. Optional iLink credentials, context tokens, recipient configuration, manifests, checkpoints, and receipts must remain in the user's private local directory and must never be committed or attached to an issue. Reports that preserve these boundaries are especially welcome.
